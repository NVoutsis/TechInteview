ΑΣΚΗΣΗ 1

1Α. ΔΗΜΙΟΥΡΓΙΑ ΣΕΙΡΑΣ ΧΑΡΤΩΝ

Πρέπει να δημιουργήσετε σειρά χαρτών (Data Driven Pages) για καθένα από τα αγροτεμάχια που σας δίνονται (ΚΑΕΚ). Η κλίμακα του κάθε χάρτη θα πρέπει να είναι υποδιαίρεση του 500. Σε κάθε χάρτη θα πρέπει να εμφανίζεται αυτοματοποιημένα τα παρακάτω:
•	το είδος του δικαιώματος (RIGHT_TYPE)
•	το ποσοστό του δικαιώματος (OWN_RIGHT_NUMER / OWN_RIGHT_DENOM)
•	το ονοματεπώνυμο του δικαιούχου (NAME / SURNAME)
•	αριθμό και ημερομηνία εγγράφου (DOC_NUM / DOC_DATE)
Επιθυμητή πληροφορία και διαμόρφωση του χάρτη (όχι απαραίτητα):
•	να φαίνεται με έντονο χρώμα το αγροτεμάχιο που αφορά ο χάρτης και με λιγότερο έντονο χρώμα τα γειτονικά αγροτεμάχια
•	να χρησιμοποιηθεί υπόβαθρο με δορυφορικές εικόνες, με προτίμηση εικόνες Bing
Παραδοτέο άσκησης: αρχείo pdf και αρχείο mxd.

1Β. ΔΗΜΙΟΥΡΓΙΑ ΧΑΡΤΟΣΥΝΘΕΣΗΣ ΜΕ ΣΥΣΧΕΤΙΣΕΙΣ ΠΙΝΑΚΩΝ
Δημιουργήστε μία νέα χαρτοσύνθεση (mxd) με συσχετίσεις (relates) που θα στηρίζονται στο πεδίο GLOBALID που υπάρχει στο Feature Class και στα Tables. Θα πρέπει να δημιουργηθεί πεδίο που θα ονομάζεται FOREIGN_KEY_xxxxxx, όπου xxxxxx το όνομα του αντίστοιχου Feature Class ή Table που αφορά το κλειδί. Η φορά των συσχετίσεων ξεκινά από το Feature Class KAEK και καταλήγει στα Tables DOCUMENTS και BENEFICIARIES.
Παραδοτέο άσκησης: αρχείο mxd.

1Γ. ΔΗΜΙΟΥΡΓΙΑ ΣΤΑΘΕΡΩΝ ΣΥΣΧΕΤΙΣΕΩΝ
Αντιγράψτε τη γεωβάση (File Geodatabase) και δημιουργήστε σταθερές συσχετίσεις (Relationship Classes) στη νέα γεωβάση μέσω του ArcCatalog, όπως έγινε στην άσκηση 1Β, αλλά δεν πρέπει να χρησιμοποιηθούν τα πεδία της άσκησης 1Β. Υπάρχουν κοινά πεδία στα δεδομένα που μπορούν να παίξουν το ρόλο του κλειδιού. Θυμηθείτε τη φορά των συσχετίσεων που φτιάξατε στην άσκηση 1Β.
Παραδοτέο άσκησης: νέα γεωβάση.
